# PRD: Daily Tracker — 투두, 잔디, 일기 통합 개인 생산성 앱

## 개요

하루의 할 일을 관리하고, 그 달성률을 깃허브 잔디처럼 시각화하며, 그날의 생각을 일기로 남길 수 있는 생산성 앱이다. 사용자별 독립 공간을 제공하며, 꾸준함을 시각적으로 확인하며 루틴을 형성하는 것이 핵심 목적이다.

## 문제 정의

- 투두 관리, 회고 일기, 달성 기록이 각각 다른 도구에 흩어져 있어 연속적인 루틴 형성이 어렵다.
- 깃허브 잔디처럼 꾸준함을 한눈에 확인할 수 있는 개인 생산성 도구가 없다.
- 지금 만들지 않으면 데이터가 축적되지 않아 장기 리뷰가 불가능하다.

## 목표

- 하루 단위로 투두 + 일기를 한 화면에서 관리할 수 있다.
- 투두 달성률이 잔디 채도로 누적되어 연간/월간 흐름을 파악할 수 있다.
- 프론트엔드 개발 없이 API + Supabase 백엔드를 완성하여 즉시 연동 가능한 상태로 제공한다.

## 비목표 (Out of Scope)

- 전체 프론트엔드 UI 구현 (잔디·툴팁 컴포넌트 제외)
- 소셜 기능 (팔로우, 댓글, 공개 피드 등)
- 모바일 앱 (네이티브)
- 알림 / 리마인더 기능
- 투두 카테고리 / 태그 / 우선순위 고급 분류

## 타깃 사용자

매일 할 일을 정리하고 하루 끝에 회고를 남기며, 장기적인 루틴 달성 기록을 시각화하고 싶은 사용자. 각 계정은 독립된 투두·일기·잔디 공간을 가진다.

## 주요 기능

### 1. 투두리스트

- 사용자는 오늘의 할 일을 추가할 수 있다.
- 사용자는 할 일을 완료 처리하거나 취소할 수 있다.
- 사용자는 날짜별 투두 목록을 조회할 수 있다.
- 사용자는 당일 전체 대비 완료 항목 수로 계산된 달성률(0–100%)을 확인할 수 있다.

### 2. 잔디(활동 히트맵) 컴포넌트

- 사용자는 연도별·월별 달성률 히트맵을 조회할 수 있다.
- 사용자는 각 셀의 채도를 통해 그날의 투두 달성률(0% / 1–33% / 34–66% / 67–99% / 100%)을 직관적으로 파악할 수 있다.
- 사용자는 셀에 마우스를 올려 해당 날짜와 달성률을 툴팁으로 확인할 수 있다.
- 잔디 그리드와 툴팁은 독립 Vue 컴포넌트로 제공되어 페이지에 바로 삽입할 수 있다.

### 3. 오늘의 일기

- 사용자는 투두리스트 옆에서 브런치 블로그 스타일의 일기를 작성할 수 있다.
- 사용자는 날짜별 일기를 조회·수정·삭제할 수 있다.
- 일기는 날짜 당 1개로 제한되며 자유 텍스트 형식이다.

### 4. 회원 인증

- 비로그인 사용자도 UID를 알면 해당 사용자의 투두·잔디·일기를 조회(read)할 수 있다.
- 투두 추가/완료, 일기 작성/수정/삭제는 로그인 상태에서만 가능하다.
- Supabase Auth 기반으로 다수 계정을 지원한다. 각 계정은 자신의 데이터만 쓰기 가능하다.

### 5. 디자인 토큰

- `#222222` 기반의 흰 바탕 + 검정 버튼 디자인 토큰을 SCSS 변수로 제공한다.
- 모든 컴포넌트는 해당 토큰을 공통으로 참조한다.

## 성공 지표

| 지표 | 현재 | 목표 |
|------|------|------|
| API 엔드포인트 완성도 | 0% | 투두/일기/잔디 CRUD 100% 구현 |
| 잔디 컴포넌트 연동 준비 | 미완성 | API 응답으로 즉시 렌더링 가능 |
| 달성률 정확도 | - | 완료 항목 수 / 전체 항목 수 × 100 (소수점 버림) |
| 인증 보안 | - | 본인 계정 데이터 외 쓰기 불가 (RLS + API 이중 검증) |

## 제약사항

- 프론트엔드는 잔디 그리드 + 툴팁 컴포넌트만 구현하며, 나머지 페이지 UI는 별도 작업이다.
- 인증은 Supabase Auth를 활용하며, 계정 생성은 Supabase 콘솔에서 직접 관리한다.
- 기술 스택: Nuxt 3, Vue 3, TypeScript, SCSS, Pinia, Supabase

## 참고자료

- 레퍼런스 UI: GitHub Contribution Graph, Brunch(brunch.co.kr) 에디터 스타일
- 잔디 채도 레벨: 0% → 무채색, 100% → 최대 채도 (5단계)
